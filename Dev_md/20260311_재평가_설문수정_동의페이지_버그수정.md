## 2. 설문 응답 수정 기능

### 배경
- 설문 완료 후 응답을 수정할 방법이 없었음
- 재평가 시 설문도 수정 가능해야 함

### 변경 파일
- `src/pages/EvalPreSurveyPage.jsx`
- `src/pages/EvalResultPage.jsx`

### 구현 내용
- `?edit=1` 쿼리 파라미터로 설문 수정 모드 진입
- 수정 모드 시:
  - 기존 응답을 DB에서 로드하여 폼에 채움
  - 자동 스킵(이미 완료) 비활성화
  - 바로 설문 단계(step 2)로 이동
  - 제출 버튼: "설문 수정 완료" (결과 페이지로 돌아감)
  - "취소" 버튼으로 결과 페이지 복귀

## 3. 동의 페이지 무한 루프 버그 수정

### 증상
- 평가자가 "개인정보 수집 동의"에서 동의 후 진행 불가
- 페이지가 무한 루프에 빠지거나, 에러 메시지 없이 멈춤

### 근본 원인 (3가지)

#### 3-1. 동의 후 설문 없을 때 무한 루프
- `handleConsent`가 설문 문항이 0개여도 `setStep(2)` 호출
- step 2가 `effectiveSteps`에 없으면 → 폴백 로직이 step 0으로 되돌림 → 무한 루프

**수정**: `questions.length === 0`이면 `navigateToEval()` 직접 호출

#### 3-2. evaluatorId null 시 무음 실패
- `handleConsent`, `handleSubmitSurvey`에서 `evaluatorId`가 null이면 그냥 return
- 사용자에게 아무 피드백 없음

**수정**: `toast.error()` 메시지 추가

#### 3-3. 모든 유효 단계 완료 후 폴백 오류
- step이 모든 `effectiveSteps`를 지난 경우 처리 로직 없음

**수정**: `step > Math.max(...effectiveSteps, -1)` 조건 추가 → `navigateToEval()` 호출

### 변경 파일
- `src/pages/EvalPreSurveyPage.jsx`

## 4. 단답형 응답 멀티 컬럼 배치

### 배경
- 설문집계 페이지의 TextResults(단답형/장문형 응답)가 1열로만 나열
- 짧은 응답(이름, 숫자)도 한 줄에 하나씩 표시되어 공간 낭비

### 변경 파일
- `src/pages/SurveyResultPage.jsx` — TextResults 컴포넌트
- `src/pages/SurveyResultPage.module.css` — textList/textItem 스타일

### 구현 내용
- 최대 글자 수 기준 자동 열 수 결정 (useMemo):
  - ≤5자 → 6열 / ≤10자 → 5열 / ≤15자 → 4열 / ≤25자 → 3열 / ≤40자 → 2열 / >40자 → 1열
- `<ul>/<li>` → CSS Grid `<div>` 변환
- 줄임표 없이 전체 텍스트 표시 (`word-break: break-word`)
- 반응형: 900px 이하 `auto-fill minmax(120px)`, 600px 이하 `auto-fill minmax(100px)`
