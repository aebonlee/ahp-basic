# 재평가 후 네비게이션 버그 수정 + 잠금 해제 안내 패널

**작업일**: 2026-03-11
**커밋**: `9c4153b`

---

## 증상

- 평가자가 결과 페이지에서 "재평가하기" 버튼 클릭 후에도 UI가 갱신되지 않음
- "설문으로 돌아가기", "일관성 평가하기" 등 네비게이션이 작동하지 않음
- 비일관성비율 탭의 행 클릭, 세부내용 탭의 페이지 이동도 비활성 상태 유지

---

## 근본 원인

### evaluators 캐시 미갱신

`handleUnlock`에서 DB 업데이트(서명 삭제, `completed=false`) 후:
- `setHasSigned(false)` → `hasSigned`만 갱신
- `useEvaluators` 훅의 캐시된 `evaluators` 배열에는 여전히 `completed: true` 유지
- `isCompleted` 메모:
  ```js
  const isCompleted = useMemo(() => {
    if (hasSigned) return true;         // false로 변경됨
    const ev = evaluators.find(...);
    return !!ev?.completed;             // 여전히 true (캐시)
  }, [evaluatorId, evaluators, hasSigned]);
  ```
- 결과: `isCompleted === true` 유지 → 모든 네비게이션/리뷰 섹션 숨김

### SignaturePanel 독립 상태

- SignaturePanel이 마운트 시 `evaluation_signatures`를 조회하여 `signed=true` 설정
- 부모에서 서명 삭제해도 SignaturePanel의 로컬 `signed` 상태 미변경
- "평가가 완료되었습니다" 메시지 유지

## 수정 내용

### 변경 파일
- `src/pages/EvalResultPage.jsx`
- `src/pages/EvalResultPage.module.css`

### 1. evaluators 캐시 강제 갱신

```jsx
// Before
const { evaluators } = useEvaluators(id);

// After
const { evaluators, fetchEvaluators } = useEvaluators(id);
```

`handleUnlock` 내에서 DB 업데이트 후 `await fetchEvaluators()` 호출:
- evaluators 목록이 DB에서 다시 조회됨
- `evaluator.completed === false` 반영
- `isCompleted`가 정확히 `false`로 변경

### 2. SignaturePanel key 전환

```jsx
<SignaturePanel key={hasSigned ? 'signed' : 'unsigned'} ... />
```

- `hasSigned` 변경 시 key가 바뀌면서 컴포넌트 리마운트
- 리마운트 후 DB 재조회 → 서명 없음 → `signed=false`
- "평가 완료" 버튼 다시 표시

### 3. 잠금 해제 안내 패널 추가

`unlocked` 상태를 추가하여 잠금 해제 직후 안내 패널 표시:

- 제목: "평가 잠금이 해제되었습니다"
- 안내: 비일관성비율 탭에서 재점검 필요 항목 확인 가능
- 안내: 수정 완료 후 하단 "평가 완료" 버튼 다시 제출
- 버튼: "쌍대비교 평가 수정" (primary) → `/eval/project/:id`
- 버튼: "설문 응답 수정" (secondary) → `/eval/project/:id/pre-survey?edit=1`

### 4. 연쇄 효과 (자동 복구)

`isCompleted === false`가 되면서:
- 상단 네비게이션: "평가로 돌아가기" + "설문 수정" 버튼 표시
- 비일관성비율 탭: 행 클릭 시 해당 페이지로 이동 가능
- 세부내용 탭: 페이지 클릭 이동 가능
- 재점검 필요 항목 섹션: 미완료/비일관성 초과 목록 표시
- SignaturePanel: "평가 완료" 버튼 표시
