# 개발일지 — PortOne V1 전환 & 장바구니 UI 완성

**날짜**: 2026-03-12
**주제**: PortOne V2 SDK → V1 (iamport) 전환, 네비바 장바구니 아이콘 추가, orderService 정리

---

## 배경

- 기존 AHP Basic 프로젝트는 PortOne V2 SDK(`@portone/browser-sdk/v2`)를 사용
- `D:\templete-ref` 템플릿이 PortOne V1 (iamport) 기반으로 안정적으로 동작 중
- 네비바에 장바구니 아이콘이 미구현 상태 — 카트 담기는 되지만 시각적 피드백 없음
- `orderService.js`에 불필요한 인메모리 폴백 코드 존재

---

## 변경 파일 (9개)

### 1. index.html — iamport V1 CDN + CSP 업데이트
- `</body>` 앞에 `<script src="https://cdn.iamport.kr/v1/iamport.js">` 추가
- CSP 정책 업데이트:
  - `script-src`: `https://cdn.iamport.kr` 추가
  - `connect-src`: `https://*.iamport.kr` 추가
  - `frame-src`: `'none'` → `https://*.iamport.kr` (결제창 iframe 허용)

### 2. src/utils/portone.js — V2 → V1 전면 교체
- **삭제**: `@portone/browser-sdk/v2` 동적 import, `STORE_ID`, `CHANNEL_KEY`
- **추가**: `VITE_IMP_CODE`, `VITE_PG_PROVIDER` 환경변수
- `getIMP()`: `window.IMP.init(IMP_CODE)` 초기화 (1회만)
- `requestPayment()`: `IMP.request_pay()` 콜백 → Promise 래핑
  - 성공: `{ paymentId: imp_uid, txId: merchant_uid }`
  - 실패: `{ code, message }`
- 데모 모드: `IMP_CODE` 미설정 시 가짜 성공 반환 유지
- `generateOrderNumber()`: AHP-YYMMDD-RANDOM6 패턴 유지

### 3. vite.config.js — external 설정 제거
- `external: ['@portone/browser-sdk/v2']` 삭제
- V2 SDK를 더 이상 사용하지 않으므로 불필요

### 4. src/components/layout/PublicNav.jsx — 장바구니 아이콘 추가
- **데스크톱**: `navActions` 영역 맨 앞에 장바구니 SVG 아이콘 + 뱃지 추가
- **모바일**: `mobileMenu` 내 divider 위에 장바구니 링크 + 뱃지 추가
- 기존 `useCart`의 `cartCount` 활용 (이미 import 되어 있었음)

### 5. src/components/layout/PublicNav.module.css — 장바구니 스타일 추가
- `.cartIconLink`: 38×38 원형 버튼, hover 시 배경 전환
- `.cartBadge`: 절대위치 빨간 뱃지 (우상단), min-width 18px
- `.mobileCartLink`: 모바일 메뉴 내 flex 레이아웃 링크
- `.mobileCartBadge`: 모바일용 빨간 뱃지 (우측 정렬)

### 6. src/pages/CheckoutPage.jsx — 이메일 폴백 보강
- 이메일 자동채우기에 `user?.identities?.[0]?.identity_data?.email` 추가
- OAuth (Google, Kakao) 로그인 시 이메일을 더 확실히 가져옴
- V1 응답 포맷이 기존 handleSubmit 로직과 호환되어 추가 수정 불필요

### 7. src/utils/orderService.js — 데드코드 정리 + 에러 처리
- **삭제**: `_memoryOrders` 인메모리 배열 및 모든 `if (!supabase)` 분기
  - `createOrder`, `getOrderByNumber`, `updateOrderStatus`, `verifyPayment`, `getOrdersByUser` 5개 함수 정리
- **추가**: `updateOrderStatus`에서 업데이트 결과가 빈 배열일 때 `UPDATE_NO_ROWS` 에러 throw

### 8. .env.example — 환경변수 문서화 업데이트
- V2 환경변수(`VITE_PORTONE_STORE_ID`, `VITE_PORTONE_CHANNEL_KEY`) → V1으로 교체
- `VITE_IMP_CODE=imp_xxxxx` / `VITE_PG_PROVIDER=kcp` 추가

### 9. .github/workflows/deploy.yml — 배포 시크릿 추가
- 빌드 단계 env에 `VITE_IMP_CODE`, `VITE_PG_PROVIDER` 시크릿 추가
- GitHub 리포지토리 Settings > Secrets에 등록 필요
