# 장바구니 & 카드 결제 구현 계획 (템플릿 참고)

## Context

`D:\templete-ref` 템플릿은 **PortOne V1 (iamport)** 기반으로 장바구니/결제를 구현하고 있다.
현재 AHP Basic 프로젝트는 **PortOne V2 SDK**를 사용하며, 네비바 장바구니 아이콘이 미구현 상태다.
템플릿 패턴에 맞춰 V1으로 전환하고, 미완성 부분을 완성한다.

## 상세 구현

### 1. index.html — iamport CDN 추가
```html
<!-- </body> 앞에 추가 -->
<script src="https://cdn.iamport.kr/v1/iamport.js"></script>
```
- CSP `script-src`에 `https://cdn.iamport.kr` 추가
- CSP `connect-src`에 `https://*.iamport.kr` 추가
- CSP `frame-src`에 `https://*.iamport.kr` 추가 (결제창이 iframe 사용)

### 2. src/utils/portone.js — V1 전환
템플릿(`D:\templete-ref\src\utils\portone.js`)과 동일한 패턴으로 전면 교체:
- 환경변수: `VITE_IMP_CODE`, `VITE_PG_PROVIDER`
- `getIMP()`: `window.IMP.init(IMP_CODE)` 초기화
- `requestPayment()`: `IMP.request_pay()` 콜백 기반 → Promise 래핑
  - 성공: `{ paymentId: response.imp_uid, txId: response.merchant_uid }`
  - 실패: `{ code: error_code, message: error_msg }`
- 데모 모드: IMP_CODE 미설정 시 가짜 성공 반환
- `generateOrderNumber()` 유지 (AHP- 프리픽스)

### 3. vite.config.js — external 제거
V2 SDK를 더 이상 사용하지 않으므로 external 설정 불필요

### 4. PublicNav.jsx — 장바구니 아이콘 렌더링
템플릿 Navbar.jsx:138-145 참고하여 `navActions` 영역에 장바구니 링크 추가:
- 데스크톱: navActions 안 (로그인/시작하기 버튼 앞)
- 모바일: mobileMenu 안에도 장바구니 링크 추가

### 5. PublicNav.module.css — 장바구니 아이콘 스타일
- `.cartIconLink` — 데스크톱 네비바 장바구니 아이콘
- `.cartBadge` — 장바구니 아이템 수 뱃지
- `.mobileCartLink` — 모바일 메뉴 장바구니 링크
- `.mobileCartBadge` — 모바일 뱃지

### 6. CheckoutPage.jsx — V1 대응
- 이메일 폴백에 `user?.identities?.[0]?.identity_data?.email` 추가
- V1 응답은 이미 `{ paymentId, txId }` 또는 `{ code, message }`이므로 기존 handleSubmit 로직과 호환

### 7. orderService.js — 코드 정리
- `_memoryOrders` 인메모리 폴백 전체 제거
- 모든 `if (!supabase)` 분기 제거
- `updateOrderStatus`에 빈 결과 에러 추가: `'UPDATE_NO_ROWS: ...'`

### 8. .env.example — 환경변수 문서화
```
VITE_IMP_CODE=imp_xxxxx
VITE_PG_PROVIDER=kcp
```

### 9. deploy.yml — GitHub Actions secrets
빌드 단계에 `VITE_IMP_CODE`, `VITE_PG_PROVIDER` 시크릿 추가

1. `npm run build` — 빌드 성공 확인 (V2 SDK 외부 의존성 없이) ✅

3. 네비바에서 장바구니 아이콘 + 뱃지 카운트 표시 확인
4. 데모 모드(IMP_CODE 미설정) 시 가짜 성공으로 OrderConfirmationPage 도달 확인
