### 1. 가입 사이트 도메인 자동 추적 (signup_domain + visited_sites)

**문제**: ahp-basic, books, competency 등 하위 사이트들이 같은 Supabase DB를 공유하지만, 회원가입/로그인 시 도메인 정보를 전달하지 않아 www.dreamitbiz.com 회원관리에서 가입사이트가 "미설정"으로 표시됨.

**원인**: 클라이언트(하위 사이트)가 도메인 정보를 DB에 전달하는 코드가 없었음. DB에 `check_user_status` RPC 함수와 `signup_domain`/`visited_sites` 컬럼은 이미 존재하지만, 이 사이트에서 호출하지 않고 있었음.

**수정 내용**:

| 파일 | 변경 |
|------|------|
| `src/utils/auth.js` | signUp `options.data`에 `signup_domain: window.location.hostname` 추가 |
| `src/contexts/AuthContext.jsx` | `onAuthStateChange`에서 `check_user_status` RPC 호출 추가 |

**auth.js 변경** (55행):

```javascript
// Before
data: { full_name: displayName },

// After
data: { full_name: displayName, signup_domain: window.location.hostname },
```

**AuthContext.jsx 변경** (82~86행):

```javascript
// onAuthStateChange 내부, loadProfile 호출 직후 추가
supabase.rpc('check_user_status', {
  target_user_id: session.user.id,
  current_domain: window.location.hostname,
}).catch(() => {}); // 실패해도 로그인 흐름에 영향 없음
```

**동작 방식**:
- 이메일 회원가입 시: `signup_domain`이 Supabase `user_metadata`에 저장됨 → `handle_new_user` 트리거가 `user_profiles`에 기록
- 모든 로그인/가입 시: `onAuthStateChange`에서 `check_user_status` RPC 호출 → `signup_domain` 미설정 시 자동 설정 + `visited_sites` 배열에 현재 도메인 추가
- OAuth(Google/Kakao) 가입도 `onAuthStateChange`를 통해 자동 처리됨
- RPC 실패 시 `.catch(() => {})`로 무시하여 로그인 흐름에 영향 없음

**DB 확인 (이미 존재)**:
- `user_profiles.signup_domain` TEXT 컬럼 — 존재 확인
- `user_profiles.visited_sites` TEXT[] 컬럼 — 존재 확인
- `check_user_status(target_user_id UUID, current_domain TEXT)` RPC — 존재 확인

### 3. 로그인 오류 수정 (Email not confirmed + Kakao OAuth)

**문제 1 — "Email not confirmed"**: 이메일 가입 후 로그인 시 "Email not confirmed" 오류 발생.

**원인**: Supabase `mailer_autoconfirm`이 `false`로 설정되어 있었고, SMTP 서버가 미설정이라 인증 메일이 발송되지 않음.

**수정**: Supabase Management API로 `mailer_autoconfirm: true` 설정 + 미인증 사용자(acebiz55@naver.com) 수동 인증 처리.

**문제 2 — Kakao OAuth 로그인 실패**: Kakao 로그인 시 인증 흐름이 완료되지 않음.

**원인**: `onAuthStateChange` 콜백 내부에서 `check_user_status` RPC를 호출하여 인증 플로우를 간섭.

**수정** (`src/contexts/AuthContext.jsx`):

```javascript
// Before: onAuthStateChange 콜백 내부에서 RPC 호출
// After: 별도 useEffect로 분리
useEffect(() => {
  if (state.user?.id) {
    supabase.rpc('check_user_status', {
      target_user_id: state.user.id,
      current_domain: window.location.hostname,
    }).catch(() => {});
  }
}, [state.user?.id]);
```
