## Phase 2: 접근성(a11y) 개선 (55→80+)

### 2-1. Modal — 포커스 트랩 + ARIA

| 속성 | 추가 내용 |
|------|----------|
| `role="dialog"` | 모달 역할 명시 |
| `aria-modal="true"` | 뒷배경 비활성화 명시 |
| `aria-labelledby="modal-title"` | 제목 연결 |
| 닫기 버튼 `aria-label="닫기"` | 스크린리더 지원 |
| 포커스 트랩 useEffect | Tab/Shift+Tab 순환, 열릴 때 첫 요소 포커스 |

### 2-2. Toast — aria-live

- 컨테이너: `aria-live="polite"`, `role="region"`
- error/warning 토스트: `role="alert"`, 나머지: `role="status"`
- 아이콘: `aria-hidden="true"`
- 닫기 버튼: `aria-label="알림 닫기"`

### 2-3. ProgressBar — role

- `role="progressbar"`, `aria-valuenow`, `aria-valuemin={0}`, `aria-valuemax={100}`, `aria-label`

### 2-4. Navbar — 테마 토글 aria

- 테마 버튼: `aria-label` 동적 ("다크 모드로 전환" / "라이트 모드로 전환")
- SA 버튼: `aria-label="슈퍼관리자 페이지"`

### 2-5. PublicNav — aria-current/expanded

- 네비 링크: `aria-current={isActive(to) ? 'page' : undefined}`
- 햄버거: `aria-expanded={mobileOpen}`, 동적 `aria-label`
- 모바일 메뉴: `aria-hidden={!mobileOpen}`
- 모바일 링크: `aria-current="page"` 추가

### 2-6. 아이콘 전용 버튼 aria-label

| 파일 | 버튼 수 | aria-label |

## Phase 3: 코드 품질 개선 (82→88)

### 3-1. SuperAdminPage — alert→toast, confirm→useConfirm

| 변경 전 | 변경 후 |
|---------|---------|
| `alert('역할 변경 실패')` | `toast.error('역할 변경 실패: ...')` |
| `alert('삭제 실패')` | `toast.error('삭제 실패: ...')` |
| `window.confirm(...)` | `await confirm({title, message, variant:'danger'})` |

- `useToast()`, `useConfirm()` 훅 + `ConfirmDialog` 컴포넌트 도입
- `toast`와 `confirm`을 props로 UsersTab/ProjectsTab에 전달

### 3-2. useCriteria — getTree 메모이제이션

| 변경 전 | 변경 후 |
|---------|---------|
| `useCallback(() => { 트리 빌드 }, [criteria])` | `useMemo(() => { 트리 빌드 }, [criteria])` + `useCallback(() => tree, [tree])` |

기존 `getTree()` 호출부 수정 불필요, 내부적으로 캐시된 tree 반환.
