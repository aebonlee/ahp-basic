### 2. [P0] ToastContext useCallback → useMemo 버그 수정

**문제**: `useCallback`은 함수 전용 메모이제이션 훅인데, 객체에 적용하고 있었음. React API 오용.

```javascript
// Before (잘못된 사용)
const toast = useCallback({
  success: (msg) => addToast(msg, 'success'),
  error: (msg) => addToast(msg, 'error'),
}, [addToast]);

// After (올바른 사용)
const toast = useMemo(() => ({
  success: (msg) => addToast(msg, 'success'),
  error: (msg) => addToast(msg, 'error'),
}), [addToast]);
```

**수정 파일**: `src/contexts/ToastContext.jsx`

### 3. [P0] 관리자 권한 DB 기반 전환

**문제**: 관리자 판별이 소스 코드에 하드코딩된 이메일 배열(`ADMIN_EMAILS`)에 의존.
번들에 포함되어 전세계에 공개되며, 관리자 추가 시 코드 수정+재배포 필요.

**수정 내용**:
- `ADMIN_EMAILS` 배열 완전 제거
- `user_profiles.role === 'admin'` DB 컬럼 기반으로 전환
- `profileLoading` 상태 추가 — 프로필 로드 완료 전 잘못된 리디렉션 방지

```javascript
// Before
const ADMIN_EMAILS = ['aebon@kakao.com', ...];
const isAdmin = isLoggedIn && ADMIN_EMAILS.includes(state.user?.email);

// After
const isAdmin = isLoggedIn && state.profile?.role === 'admin';
```

**수정 파일**: `src/contexts/AuthContext.jsx`

### 17. PublicNav / PublicFooter / PublicLayout 공통 컴포넌트 추출

**배경**: HomePage에 인라인으로 작성된 네비게이션과 푸터를 공통 컴포넌트로 분리하여, 새로 추가되는 공개 페이지들과 공유.

**구현 내용**:

| 컴포넌트 | 파일 | 설명 |
|----------|------|------|
| `PublicNav` | `src/components/layout/PublicNav.jsx` + `.module.css` | 고정 네비, 라우트 링크, active 하이라이트, 모바일 햄버거 메뉴 |
| `PublicFooter` | `src/components/layout/PublicFooter.jsx` + `.module.css` | 다크블루 푸터, 페이지 링크 3개, 저작권 |
| `PublicLayout` | `src/components/layout/PublicLayout.jsx` + `.module.css` | Nav + children + Footer 래퍼 |

**네비게이션 구조 변경**:
```
Before: [AHP Basic]  기능(#features)  프로세스(#process)  시작하기(#start)       [로그인] [시작하기]
After:  [AHP Basic]  AHP 소개(/about)  주요 기능(/features)  이용 가이드(/guide)  [로그인] [시작하기]
```

- 기존 앵커 링크(`#features`, `#process`, `#start`) → 별도 페이지 라우트(`/about`, `/features`, `/guide`)
- `useLocation()` 기반 현재 경로 하이라이트 (active 상태)
- 모바일(768px 이하): 햄버거 메뉴로 전환, 링크 + 로그인/시작하기 버튼 표시
- 로그인 상태 시: 로그인/시작하기 대신 "대시보드" 버튼 표시
