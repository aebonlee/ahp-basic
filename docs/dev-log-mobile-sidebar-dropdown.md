# 개발일지: 모바일 2차 메뉴 드롭다운 전환

**날짜**: 2026-03-13
**작업 유형**: UX 개선 (모바일 내비게이션)
**주제**: ProjectSidebar 모바일 수평 스크롤 → 드롭다운 토글 방식 전환
**상태**: 완료

---

## 배경

모바일(768px 이하)에서 프로젝트 단계 메뉴(13개 항목)가 **수평 스크롤 1줄**로 나열되어 있어 다음 문제가 있었다:

1. 원하는 단계를 찾기 위해 좌우로 스크롤해야 함
2. 현재 어떤 단계에 있는지 한눈에 파악 어려움
3. 하위 메뉴(통계 분석 11개, AI분석 4개)가 수평 레이아웃에서 사실상 사용 불가
4. 평가자/연구자가 모바일에서 정보 확인 시 불편

---

## 해결 방안: 드롭다운 토글 방식

### 데스크톱 (변경 없음)
- 기존 세로 사이드바 유지 (200px 좌측)
- 접기/펼치기 토글 버튼 유지

### 모바일 (768px 이하) — 새로운 패턴

```
┌─────────────────────────────┐
│ ☰ 프로젝트 목록             │
│ 프로젝트명                  │
│ 7. 집계 결과              ▾ │  ← 현재 단계 표시 토글 바
├─────────────────────────────┤
│  ① 모델 구축               │  (탭하면 세로로 전체 메뉴 펼침)
│  ② 브레인스토밍            │
│  ③ 모델 확정               │
│  ...                        │
│  ⑪ 통계 분석        ▶      │  ← 하위 메뉴 아코디언
│    1. 기술통계              │
│    2. 독립표본 T검정        │
│  ⑫ AI분석도구활용   ▶      │
│  ⑬ 문자 이력               │
└─────────────────────────────┘
```

**장점:**
- 현재 단계가 항상 보임 (토글 바에 "7. 집계 결과" 형태)
- 탭 한 번으로 전체 메뉴 확인 (스크롤 불필요)
- 하위 메뉴(통계/AI)가 아코디언으로 자연스럽게 동작
- 메뉴 선택 시 자동 닫힘 (화면 공간 절약)
- `max-height` 트랜지션으로 부드러운 애니메이션

## 수정 파일 (2개)

### 1. `src/components/layout/ProjectSidebar.jsx`

**추가 상태:**
- `mobileNavOpen`: 모바일 드롭다운 열림/닫힘
- `currentStep`: 현재 활성 단계 계산 (토글 바 표시용)

**추가 함수:**
- `handleMobileNavigate(path)`: navigate 후 `mobileNavOpen = false` (자동 닫힘)

**JSX 추가:**
- 모바일 토글 바 (`mobileToggle`): 현재 단계명 + ▾ 화살표
- `aria-expanded` 접근성 속성 추가
- nav에 `navMobileOpen` 조건부 클래스 추가

### 2. `src/components/layout/ProjectSidebar.module.css`

**데스크톱:**
- `.mobileToggle { display: none }` — 데스크톱에서 숨김

**모바일 (768px 이하):**
- `.mobileToggle` 표시: primary-bg 배경, 현재 단계명 + 회전 화살표
- `.nav`: `flex-direction: column` 유지, `max-height: 0 → 800px` 트랜지션
- `.navMobileOpen`: overflow-y auto, border-top 구분선
- `.menuItem`: 세로 배치, border-bottom 구분선, 적절한 패딩
- `.subMenu` / `.subItem`: 모바일 패딩 조정
- 기존 수평 스크롤 코드 (`flex-direction: row`, `overflow-x: auto`) 완전 제거
