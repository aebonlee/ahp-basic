# 공동 연구자 결과 공유 링크 (토큰 방식) 구현 계획

## Context
프로젝트 소유자가 "결과 공유 링크"를 생성하면, 해당 링크를 받은 공동 연구자가 **로그인 없이** 집계 결과를 읽기 전용으로 볼 수 있게 한다.

## 구현 범위

### 1. DB 마이그레이션 — `supabase/migrations/023_result_share_token.sql`

- `projects` 테이블에 `result_share_token UUID` 컬럼 추가 (nullable, unique)
- RPC 함수 `get_shared_result(p_token UUID)` (SECURITY DEFINER):
  - 토큰으로 project_id를 찾고, 결과 표시에 필요한 모든 데이터를 한번에 JSON으로 반환
  - 반환: project info, criteria, alternatives, evaluators(이름만), pairwise_comparisons, direct_input_values
  - 로그인 불필요 (anon 접근 가능)

### 2. AdminResultPage 수정 — `src/pages/AdminResultPage.jsx`

- 상단에 "결과 공유" 버튼 추가
- 클릭 시:
  - 토큰이 없으면 UUID 생성 → `projects.result_share_token` UPDATE
  - 이미 있으면 기존 토큰 사용
  - 공유 URL을 클립보드에 복사: `{origin}/#/shared/result/{token}`
- "공유 해제" 버튼: 토큰을 null로 UPDATE하여 비활성화

### 3. 공유 결과 페이지 — `src/pages/SharedResultPage.jsx` (새 파일)

- 라우트: `/shared/result/:token` (가드 없음, 누구나 접근 가능)
- RPC `get_shared_result(token)` 호출하여 데이터 로드
- AdminResultPage와 동일한 결과 컴포넌트 렌더링 (읽기 전용):
  - ComprehensiveChart, ResultSummary, DetailView, ConsistencyTable
- **제외**: EvaluatorWeightEditor (가중치 조정 불가), ExportButtons
- 프로젝트명 표시, "읽기 전용 공유 결과" 안내 배너

### 4. 라우트 등록 — `src/App.jsx`

- `/shared/result/:token` 라우트 추가 (가드 없음)
- SharedResultPage lazy import

## RPC 설계: `get_shared_result`

핵심 전략: 개별 테이블 RLS를 건드리지 않고, 단일 RPC 함수(SECURITY DEFINER)로 필요한 데이터만 선별하여 JSON 반환

```
입력: p_token UUID
출력: JSON {
  project: { id, name, eval_method },
  criteria: [...],
  alternatives: [...],
  evaluators: [{ id, name }],  -- email/phone 제외 (개인정보 보호)
  comparisons: [...],
  direct_inputs: [...]
}
토큰 불일치 시: NULL 반환
```

## 검증
- 소유자가 AdminResultPage에서 공유 링크 생성/복사 가능한지 확인
- 비로그인 사용자가 공유 링크로 접속하여 결과를 볼 수 있는지 확인
- 토큰 없거나 잘못된 경우 에러 표시 확인
- 공유 해제 후 기존 링크가 무효화되는지 확인
