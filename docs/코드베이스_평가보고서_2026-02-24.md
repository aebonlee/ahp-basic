### 1.2 라우팅 구조 평가

`HashRouter`를 사용하며 GitHub Pages 정적 배포에 최적화되어 있다.

**잘 된 점:**
- 관리자 경로(`/admin/*`)와 평가자 경로(`/eval/*`)의 명확한 분리
- `ProtectedRoute`로 인증 게이트웨이 통일
- `*` 경로의 홈으로 fallback 처리

**문제점:**
- `AdminGuard` 컴포넌트가 라우터에서 전혀 사용되지 않음 (데드 코드)
- 관리자 권한 검증이 하드코딩된 이메일 목록에만 의존
- `/register`와 `/signup`이 같은 컴포넌트를 가리키는 중복 라우트

### 1.3 관심사 분리 평가

| 계층 | 평가 |
|------|------|
| 계산 엔진 (`lib/`) | **우수**: 순수 함수, React 미의존 |
| 데이터 접근 (`hooks/`) | **양호**: Supabase 호출이 훅에 캡슐화 |

| 페이지 (`pages/`) | **일부 문제**: 일부 페이지가 너무 많은 책임 |

## 2. 데이터 레이어 및 상태 관리

### 2.1 훅 설계 패턴

**잘 설계된 훅들:**
- `useCriteria.js`/`useAlternatives.js`: `useRef`로 stale closure 방지, cascade 삭제 BFS 구현
- `useConfirm.js`: Promise + useRef 기반 명령형 다이얼로그

**문제가 있는 훅들:**
- `useEvaluators.js`: 에러 상태(`error`)가 없어 조용히 실패
- `useProjects.js`: ESLint 규칙을 명시적으로 우회하는 의존성 배열

### 2.2 직접 DB 호출 위치 (훅 외부)

다음 컴포넌트들이 훅을 거치지 않고 `supabase`를 직접 호출:
- `WorkshopPage.jsx`: `loadProgress()` 함수
- `EvaluatorMainPage.jsx`: `loadAssignedProjects()` 내 3개의 직접 쿼리
- `InviteLandingPage.jsx`: `checkInvite()` 내 직접 쿼리
- `EvalPreSurveyPage.jsx`: `checkCompleted()` 직접 호출
- `PairwiseRatingPage.jsx`: `checkSurvey()` 내 직접 쿼리

## 3. 보안 평가

### 3.1 양호한 점
- Supabase RLS가 `is_project_owner()`, `is_project_evaluator()` 헬퍼로 체계적 설계
- React의 기본 XSS 방지 (`dangerouslySetInnerHTML` 미사용)

### 3.2 취약점

**(1) 관리자 이메일 하드코딩 [중간 위험]**
```javascript
const ADMIN_EMAILS = ['aebon@kakao.com', ...]; // 번들에 공개
```

**(2) Supabase URL 하드코딩 [중간 위험]**
```javascript
const supabaseUrl = import.meta.env.VITE_SUPABASE_URL || 'https://...';
```

**(3) 관리자 라우트 UI 수준 보호 부재**
- `/admin/*`에 `ProtectedRoute`만 사용, `AdminGuard` 미적용

**(4) 평가자 인증 부재**
- `/eval/project/:id`에 해당 프로젝트 평가자인지 확인하는 가드 없음

## 4. 성능 평가

- `ahpBestFit.js`의 O(n²×19) 동기 계산: Web Worker 오프로드 필요
- `reorderQuestions`의 N회 순차 DB 호출: 배치 upsert로 변경 필요
- `EvaluationProvider`가 앱 전체를 감쌈: `/eval/*` 범위로 축소 필요
- 코드 분할 미적용: `recharts`, `xlsx` lazy loading 필요

## 7. 결론

이 코드베이스는 AHP 연구 도구로서 충분한 완성도를 갖추고 있다. 핵심 AHP 알고리즘의 정확성, 직관적인 10단계 워크플로우, 실시간 워크숍 기능이 특히 돋보인다. 프로덕션 배포 전 보안 강화(P0)와 에러 처리 표준화(P1)가 우선 과제이며, 나머지는 점진적 개선이 가능하다.
