## 2. 신규 파일 (8개)

### DB 마이그레이션
- `supabase/migrations/004_survey_tables.sql`
  - `projects` 테이블에 `research_description`, `consent_text` 컬럼 추가
  - `survey_questions` 테이블: 설문 질문 (7가지 유형 지원)
  - `survey_responses` 테이블: 설문 응답 (프로젝트×평가자×질문 UNIQUE)
  - `consent_records` 테이블: 동의 기록
  - RLS 정책: 프로젝트 소유자 CRUD + 평가자 본인 데이터 INSERT/SELECT

### 데이터 Hook
- `src/hooks/useSurvey.js`
  - `useSurveyQuestions(projectId)` — 질문 CRUD (fetch, add, update, delete, reorder)
  - `useSurveyResponses(projectId)` — 응답 fetch/submit (전체/평가자별/질문별)
  - `useSurveyConfig(projectId)` — research_description, consent_text 읽기/저장
  - `useConsentRecords(projectId)` — 동의 기록 관리

### 설문 설계 페이지 (관리자)
- `src/pages/SurveyBuilderPage.jsx`
- `src/pages/SurveyBuilderPage.module.css`
  - 3개 섹션: 연구 소개 / 개인정보 동의서 / 인구통계학적 질문
  - 질문 유형: 단답형, 장문형, 객관식, 체크박스, 드롭다운, 숫자, 리커트 척도
  - 질문 카드: 텍스트 편집, 유형 선택, 옵션 편집기, 필수 토글, 순서 이동, 삭제
  - 자동 저장 (blur/change 시)

### 설문 집계 페이지 (관리자)
- `src/pages/SurveyResultPage.jsx`
- `src/pages/SurveyResultPage.module.css`
  - 응답 현황: N명 응답 / N명 배정
  - 질문별 결과:
    - 단답형/장문형 → 텍스트 리스트
    - 객관식/드롭다운/체크박스/리커트 → Recharts BarChart
    - 숫자 → 통계 요약 (최소/최대/평균/중앙값)

### 평가자 사전설문 페이지
- `src/pages/EvalPreSurveyPage.jsx`
- `src/pages/EvalPreSurveyPage.module.css`
  - 3단계 플로우: 연구 소개 → 동의서 → 설문 응답
  - 스텝 인디케이터 (활성/완료 상태)
  - 질문 유형별 렌더링 (7가지)
  - 필수 응답 검증
  - 완료 시 평가 페이지로 자동 이동

## 4. 지원하는 질문 유형 (7가지)

| 유형 | DB값 | 평가자 UI | 집계 표시 |
|------|------|----------|----------|
| 단답형 | short_text | `<input text>` | 텍스트 리스트 |
| 장문형 | long_text | `<textarea>` | 텍스트 리스트 |
| 객관식 | radio | 라디오 버튼 그룹 | BarChart |
| 체크박스 | checkbox | 체크박스 그룹 | BarChart |
| 드롭다운 | dropdown | `<select>` | BarChart |
| 숫자 | number | `<input number>` | 통계 요약 |
| 리커트 척도 | likert | 1~5 수평 라디오 | BarChart |

## 5. DB 스키마

### survey_questions
```sql
id UUID PK
project_id UUID FK → projects(id)
question_text TEXT
question_type TEXT (short_text|long_text|radio|checkbox|dropdown|number|likert)
options JSONB (선택지 배열)
required BOOLEAN
sort_order INTEGER
created_at TIMESTAMPTZ
```

### survey_responses
```sql
id UUID PK
project_id UUID FK → projects(id)
evaluator_id UUID FK → evaluators(id)
question_id UUID FK → survey_questions(id)
answer JSONB
created_at TIMESTAMPTZ
UNIQUE(project_id, evaluator_id, question_id)
```

### consent_records
```sql
id UUID PK
project_id UUID FK → projects(id)
evaluator_id UUID FK → evaluators(id)
agreed BOOLEAN
agreed_at TIMESTAMPTZ
UNIQUE(project_id, evaluator_id)
```

### projects (추가 컬럼)
```sql
research_description TEXT DEFAULT ''
consent_text TEXT DEFAULT ''
```
