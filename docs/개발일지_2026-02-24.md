## 주요 변경 사항

### 1. [Critical] 평가 데이터 저장 불가 버그 수정 — evaluator_id FK 불일치

**문제**: `pairwise_comparisons.evaluator_id`는 `evaluators.id` (평가자 레코드 UUID)를 참조하는 FK인데,
코드에서 `user?.id` (auth.users.id, 인증 UUID)를 전달하고 있었음.
결과적으로 모든 쌍대비교 저장이 FK 제약조건 위반으로 **무조건 실패**하여 평가 데이터가 DB에 저장되지 않았음.

**원인 분석**:
- `evaluators` 테이블: `id` (PK, 자체 UUID), `user_id` (auth.users.id 참조)
- `pairwise_comparisons` 테이블: `evaluator_id` → `evaluators.id` 참조 (FK)
- 코드: `evaluatorId={user?.id}` → auth UUID를 evaluators.id로 잘못 전달

**수정 내용** (4개 페이지 + Context):
| 파일 | 변경 |
|------|------|
| `PairwiseRatingPage.jsx` | `useEvaluators(id)` 추가, `evaluators.find(e => e.user_id === user?.id)?.id`로 올바른 ID 조회 |
| `DirectInputPage.jsx` | 동일 패턴 적용 |
| `EvalResultPage.jsx` | 동일 패턴 적용 |
| `SignaturePanel.jsx` | `.eq('user_id', evaluatorId)` → `.eq('id', evaluatorId)` 수정 |
| `EvaluationContext.jsx` | `loadProjectData(projectId, evaluatorId)` 시그니처 변경, evaluator별 비교데이터 필터링 추가 |

### 2. 평가 진행도 표시 개선

**문제**: 사용자가 "0/12 진행도 표시도 되게 해줘"라고 요청. 기존 진행바는 페이지 진행률만 표시하고 있었고,
evaluator_id 버그로 인해 완료 항목이 항상 0으로 표시됨.

**수정 내용**:
- `EvaluationProgress.jsx`: 진행바가 실제 셀 완료율을 반영하도록 변경
- 표시 형식: "평가 완료: **3/12** 항목 (25%)"
- 스타일 개선: 배경 카드, 더 두꺼운 진행바 (8→10px), 그라디언트 색상
