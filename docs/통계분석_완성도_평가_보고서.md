## 1. 수학/통계 정확성 — 치명적 이슈 4건

### [P0] tCDF()가 양측 p값이 아님

**파일**: `src/lib/statsDistributions.js:164-168`

```javascript
export function tCDF(t, df) {
  const x = df / (df + t * t);
  const p = regularizedBeta(x, df / 2, 0.5);
  return p; // ← 실제로는 양측이 아닌 regularizedBeta 값 그대로 반환
}
```

**문제**: `I_x(df/2, 0.5)` 공식은 `P(|T| > |t|)` 를 직접 계산하므로 이미 양측 p값이 맞음. 단, 이 공식이 정확히 동작하려면 `regularizedBeta()` 구현이 완벽해야 함.

**실제 검증 필요**: t=2.0, df=10 → 기대 p≈0.0736 vs 실제 출력값 비교

**영향**: 독립표본 T검정, 대응표본 T검정, 상관분석, 회귀분석 — 모든 t-기반 검정의 유의성 판단

---

### [P0] pearsonR()에서 완전상관(r=±1) 시 0 나누기

**파일**: `src/lib/statsEngine.js:406`

```javascript
const t = r * Math.sqrt((n - 2) / (1 - r * r));
// r = ±1이면 (1 - r*r) = 0 → Infinity → NaN 전파
```

**영향**: 상관분석, 크론바흐 알파 (항목-총점 상관에서 pearsonR 호출)

**수정안**: `if (Math.abs(r) >= 0.9999) return { r, p: 0 };` 가드 추가

---

### [P1] normalCDF() 급수 수렴 불안정

**파일**: `src/lib/statsDistributions.js:193-203`

```javascript
let sum = 0, term = z;
for (let i = 3; sum + term !== sum; i += 2) {
  sum += term;
  term = term * z * z / i;
}
```

**문제**: 이 급수는 z가 클 때 수렴 느리고, 부동소수점 비교 `sum + term !== sum`이 불안정. 현재 `normalCDF`는 코드 내에서 직접 호출되지 않으므로 즉시 영향은 없으나, 향후 사용 시 위험.

---

### [P1] regularizedBeta()의 경계값 불안정

**파일**: `src/lib/statsDistributions.js:36-37`

```javascript
const lnPre = a * Math.log(x) + b * Math.log(1 - x) - Math.log(a) - ...
```

**문제**: x ≈ 0 또는 x ≈ 1 일 때 `Math.log(0)` = `-Infinity` → NaN 전파. 극단적 t/F/χ² 값에서 p값이 NaN 반환 가능.

## 2. 코드 품질/안정성 — 주요 이슈 8건

### [P1] 유틸리티 함수 빈 배열 미처리

**파일**: `src/lib/statsEngine.js:8-9`

```javascript
function mean(arr) {
  return arr.reduce((s, v) => s + v, 0) / arr.length;
  // arr = [] → 0 / 0 = NaN → 이후 모든 계산 NaN 전파
}
```

`mean()`, `variance()`, `std()`, `median()` 모두 빈 배열 가드 없음. `descriptiveStats()`는 `n === 0` 체크하지만, 다른 분석 함수에서 직접 호출 시 크래시 가능.

---

### [P1] handleRun()에 try-catch 없음

**파일**: `src/pages/StatisticalAnalysisPage.jsx:60-132`

분석 실행 중 어떤 이유로든 예외 발생 시 (NaN, 빈 배열, 타입 에러 등) 페이지 전체 화이트스크린. ErrorBoundary가 App 레벨에 있으나, 사용자 경험상 분석 에러는 Toast로 표시하는 것이 적절.

---

### [P2] getGroupedNumericValues()에서 checkbox 응답 join 문제

**파일**: `src/hooks/useStatisticalAnalysis.js:79-82`

```javascript
group: Array.isArray(v) ? v.join(', ') : String(v ?? ''),
```

체크박스 응답 `["A", "B"]` → `"A, B"` 문자열로 변환. `["B", "A"]`와 다른 그룹으로 분류됨. 그룹변수로 checkbox 타입 사용 시 잘못된 그룹 분할.

---

### [P2] VariableSelector isValid()에서 빈 문자열 통과

**파일**: `src/components/statistics/VariableSelector.jsx:97`

```javascript
if (!val) return false; // "" (빈 문자열)은 falsy → 실제로 차단됨
```

재확인 결과: JavaScript에서 `!""` = `true`이므로 빈 문자열은 차단됨. **이슈 아님**.

---

### [P2] mode() 함수에서 NaN 키 문제

**파일**: `src/lib/statsEngine.js:28-42`

```javascript
for (const v of arr) freq[v] = (freq[v] || 0) + 1;
```

`freq[NaN]`은 객체 키가 `"NaN"` 문자열로 변환됨. 입력 데이터에 NaN이 포함되면 잘못된 최빈값 산출. 단, `getNumericValues()`에서 NaN 필터링하므로 실제 영향은 낮음.

---

### [P2] 상관행렬 대칭 중복 계산

**파일**: `src/lib/statsEngine.js:342`

```javascript
const { r, p } = pearsonR(vars[i].values, vars[j].values);
```

(i,j)와 (j,i) 양쪽 모두 계산. 변수 10개면 90회 → 45회로 절반 줄일 수 있음. 성능 이슈.

### [P3] 크론바흐 알파 — 제외 응답자 미보고

**파일**: `src/hooks/useStatisticalAnalysis.js:124-129`

모든 문항에 응답하지 않은 평가자를 조용히 제외. 사용자에게 "N명 중 M명만 분석에 포함됨" 안내 없음.

### [P3] 독립T검정 — 2개 초과 그룹 처리

**파일**: `src/pages/StatisticalAnalysisPage.jsx:84-93`

그룹이 3개 이상이면 첫 2개만 사용. 사용자에게 "3개 이상 그룹 감지, 첫 2개만 비교합니다" 경고 없음.
