# 개발일지: SMS 모달에 전화번호 직접 입력 기능 추가

**날짜:** 2026-03-17
**작업자:** Claude Opus 4.6

## 개요

프로젝트 평가자 관리 페이지의 SMS 발송 모달(`SmsModal`)에 전화번호를 직접 입력하여
발송할 수 있는 기능을 추가했다. 기존에는 등록된 평가자 목록에서만 수신자를 선택할 수 있었으나,
이제 평가자 목록 외의 번호에도 메시지를 보낼 수 있다.

또한 수신자 목록이 길어질 때 모달 전체에 세로 스크롤이 생기는 문제를 수정했다.

## 변경 파일

| 파일 | 변경 내용 |
|------|-----------|
| `src/components/admin/SmsModal.jsx` | 직접 번호 입력 상태/파싱/발송 로직 추가 |
| `src/components/admin/SmsModal.module.css` | 직접 입력 UI 스타일 + 수신자 목록 스크롤 제한 |

## 상세 변경 사항

### SmsModal.jsx

1. **직접 입력 상태 추가**
   - `directPhones`: 전화번호 텍스트 입력값
   - `directOpen`: 토글 열림/닫힘 상태
   - `parsedDirectPhones`: 유효 번호 배열 (쉼표/줄바꿈 구분, 하이픈/공백 제거, `0XX` 형식 검증)

2. **수신자 카운트 통합**
   - `selectedCount` = 평가자 선택 수 + 직접 입력 번호 수
   - 발송 버튼에 통합 카운트 표시

3. **발송 로직 통합**
   - `handleSend()`에서 평가자 수신자 + 직접 입력 수신자를 합쳐서 `sendSmsBulk()` 호출
   - 직접 입력 번호는 `name: ''`으로 전달, 결과에서 번호로 표시

4. **접이식 UI**
   - "번호 직접 입력" 토글 버튼 (수신자 목록 하단)
   - 열면 textarea + 유효 번호 카운트 표시
   - 접은 상태에서도 입력된 번호 수 배지 표시

### SmsModal.module.css

1. **스크롤 수정**: `.recipientList`에 `max-height: 320px; overflow-y: auto` 추가
   - 수신자 목록이 길어도 모달 전체 스크롤 방지
2. **직접 입력 섹션 스타일**: `.directSection`, `.directToggle`, `.directInput`, `.directHint` 추가
   - 기존 녹색 테마와 일관된 디자인

## UI 구조

```
SMS 발송 모달 (980px)
├── 왼쪽: 수신자 선택
│   ├── 필터 칩
│   ├── 평가자 목록 (max-height 320px 스크롤)
│   ├── 페이지네이션
│   └── ▸ 번호 직접 입력 (접이식)  ← 신규
│       ├── textarea (쉼표/줄바꿈 구분)
│       └── 유효 번호 카운트
└── 오른쪽: 메시지 입력
    ├── 템플릿 탭
    ├── 메시지 textarea
    └── 바이트 카운터
```
