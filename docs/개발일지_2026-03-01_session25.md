# 개발일지 — 세션 25

**날짜**: 2026-03-01
**목표**: 컬러 아이덴티티 개선 — 밋밋한 모노톤에서 영역별 컬러감으로 전환

---

## 배경

기존 사이트는 다크블루(`#0f2b5b`) 하나만 모든 곳에 사용하여 아이콘, 카드, 버튼, 통계값이 전부 동일 색상이었음. AHP 워크플로우 4단계에 각각 고유 컬러 아이덴티티를 부여하여 시각적 구분감과 전문성을 높임.

## 컬러 팔레트 (4색 — AHP 워크플로우 매핑)

| # | 기능 | 색상 | 주색 |
|---|------|------|------|
| 1 | 계층 모델 구축 | Blue | `#2563eb` |
| 2 | 쌍대비교 평가 | Violet | `#7c3aed` |
| 3 | 다수 평가자 집계 | Teal | `#0891b2` |
| 4 | 결과 분석 | Emerald | `#059669` |

## 작업 내용

### 1. CSS 변수 추가 (`variables.css`)

- `:root`에 12개 feature 색상 변수 추가 (`--color-feature-1` ~ `--color-feature-4`, 각각 주색/배경/라이트)
- `[data-theme="dark"]`에 동일 12개 다크 테마 오버라이드 추가

### 2. HomePage 컬러 적용 (`HomePage.module.css`)

| 영역 | Before | After |
|------|--------|-------|
| Hero 글로우 | 단일 블루 radial | 멀티컬러(blue+violet+teal) 3레이어 |
| 4개 기능 카드 아이콘 | 전부 `#f0f4fa + #0f2b5b` | 각각 Blue/Violet/Teal/Emerald |
| 카드 하단 언더라인 | 전부 `navy→blue` | 각 카드 자체 컬러 그라디언트 |
| 카드 호버 보더 | 전부 `#bfdbfe` | 각 컬러 200톤 |
| 워터마크 번호 | 전부 `rgba(navy, 0.04)` | 각 컬러 `rgba(color, 0.06)` |
| Stats Bar 값 | 전부 `#0f2b5b` | 4개 각각 feature 컬러 |
| Process 스텝 번호 | 전부 `#60a5fa` | 각 컬러 300톤 |
| Process 스텝 라인 | 전부 `rgba(white, 0.15)` | 각 컬러 패스텔 30% |
| CTA 글로우 | 단일 블루 | blue+violet 듀얼 그라디언트 |

### 3. FeaturesPage 컬러 적용 (`FeaturesPage.module.css`)

- 4개 기능 번호 박스: 각 feature 컬러 bg+text
- 기능별 태그 pills: 각 feature 컬러 bg+text
- 구분선 `<hr>`: 각 feature 컬러 → transparent 그라디언트
- 4개 부가기능 아이콘: 각 feature 컬러
- 부가기능 호버 보더: 각 컬러 200톤

### 4. AboutPage 컬러 적용 (`AboutPage.module.css`)

- 4개 개념 카드 아이콘 (AB/CR/EV/GM): 각 feature 컬러 bg+text
- 카드 하단 언더라인: 각 컬러 자체 그라디언트
- 카드 호버 보더: 각 컬러 200톤
- 3개 플랫폼 아이콘: Blue/Violet/Teal
- 플랫폼 호버 보더: 각 컬러 200톤

### 5. AdminDashboard 컬러 적용 (`AdminDashboard.module.css`)

- 스텝 번호 원형 보더: 워크플로우 단계별 `rgba(color, 0.2)`
- 스텝 1~3(모델 구축): Blue 유지
- 스텝 4(평가자 관리): Violet
- 스텝 5(실시간 워크숍): Teal
- 스텝 6~8(결과 분석): Emerald
- Done 상태(초록색), 상태 뱃지: 변경 없음
